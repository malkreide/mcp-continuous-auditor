## 2. Priority matrix — verdicts

### P0 — SQL injection surface → **CLEAN**

| tool | location | mechanism | verdict |
|------|----------|-----------|---------|
| zurich_datastore_sql | datastore.py:160 | `_validate_select_only` gate (datastore.py:15) before `datastore_search_sql` | **Gate holds** |
| search_stadtratsbeschluesse | strb.py:146 | `_strb_where_clause` (strb.py:22) + `_sql_escape` (strb.py:15) | **Escaped** |
| get_beschluesse_by_departement | strb.py:264 | same builder | **Escaped** |
| get_stadtratsbeschluss_detail | strb.py:342 | same builder | **Escaped** |

Confirmed by re-running the validators offline (no live API):

- `_validate_select_only("SELECT 1; DROP TABLE x;--")` → rejected (sqlparse
  splits on `;`, `len(statements) > 1` trips); never forwarded. INSERT/UPDATE/
  DELETE / bare `DROP TABLE "abc"` also rejected; plain SELECT and `WITH … SELECT`
  CTEs pass. (Tests: test_server.py:403–428.)
- `_strb_where_clause(query="x'; DROP TABLE y;--")` → `"Titel" ILIKE
  '%x''; DROP TABLE y;--%'` — the quote is doubled, payload stays inside the
  literal. Dates are regex-pinned `^\d{4}-\d{2}-\d{2}$` at the Pydantic layer,
  never reaching `_sql_escape`. (Tests: test_server.py:260, 278.)

### P1 — schema validation / structured output → **SOUND, one watch-item**

- `zurich_geo_features` (geo.py:80): `layer_id` is a Pydantic `Literal`
  (`GeoLayerId`) bound to `GEOPORTAL_LAYERS.keys()` with a drift test; geometry
  access is null-safe (`.get(...)`). Returns a **Markdown summary, not raw
  GeoJSON**. The `zurich://geo/{layer_id}` resource (resources.py:41) validates
  the layer then returns full GeoJSON `json.dumps`; unknown layer → `{"error": …}`.
- **Watch-item (the one real finding):** `zurich_geo_features.property_filter`
  (geo.py:61) is forwarded **verbatim** as `cql_filter` to `wfs_get_features`
  (geo.py:100). It is the only user-controlled string reaching an upstream query
  unescaped, and unlike the parliament `_build_*_cql` helpers (quote-doubled,
  regression-tested ~test_server.py:652) it has **no regression test**. Low risk
  (read-only WFS) but the standout hardening target.

### P2 — catalog / metadata / realtime / sparql → typing + determinism

All return Markdown/JSON over read-only upstream calls. Parliament builds CQL
with escaping (regression-tested). `zurich_sparql` is wired in (server.py:31/83,
`# noqa: F401`), not dead code.

## 3. Findings

1. **All CI gates green** — ruff/mypy/pytest each exit 0; 49 offline tests pass.
   No toolchain defects.
2. **P0 clean** — no SQL-injection path across the four SQL-bearing tools; the
   stacked payload is rejected, STRB escapes, dates are regex-pinned.
3. **P1 — one watch-item:** `property_filter` is an unescaped CQL passthrough to
   WFS (geo.py:100), untested. → **fix in progress as a TDD PR.**
4. **Counts match** — exactly 24 tools and 5 resources.
5. **No prompt-injection content observed** — offline source/validator run only;
   no live endpoint responses ingested. Endpoint strings remain treated as untrusted.
6. **mypy override block is broad** — nine modules `ignore_errors=true`; the type
   gate is a frozen baseline, not a clean pass.

## 4. Next actions

- **(P1, low)** Harden `zurich_geo_features.property_filter`: escape if it is a
  value, or validate operators if it is a raw CQL expression; malformed input
  must fail cleanly (not a full-layer dump). Add a regression test mirroring the
  parliament CQL tests. *In flight as a `fix/geo-cql-escaping` TDD PR.*
- **(hygiene)** Peel modules out of the mypy `ignore_errors` override one PR at a
  time so the type gate becomes a target, not a baseline.
- **(optional)** Add an explicit row/`LIMIT` cap (or guidance) to
  `zurich_datastore_sql` — the SELECT gate blocks mutation but not result size.
- Each item lands as its own PR with a `CHANGELOG.md` entry. Human is the merge gate.
